2. Completed work — detail

Fix #10 — Doubled-brand titles Done

  • Root cause: layout always appended | Sweet Home while some landing pages already included the brand in the title.
  • Fixed title template / head logic so the brand appears once.
  • Validation: 21/21 affected URLs PASS live (15 Jul 2026).

Fix #11 — Language-correct titles and H1 Done

  • German default pages (e.g. /about, /contact, /cookies) now use German <title> and H1.
  • EN and ES locales also localized for static pages, services, and projects list.
  • Titles set in route handlers via t() (layout renders <title> before page body).
  • Validation: Live DE / EN / ES checks PASS (15 Jul 2026).

Fix #12 — Missing meta descriptions Done

  • Root cause: SEO head partials failed silently (variable shadowing / TDZ), so EN/ES home and projects pages emitted no description; DE home fell back to English copy.
  • Repaired head partials; added localized meta for home, projects, about, contact, cookies.
  • Validation: 12/12 core URLs PASS live (15 Jul 2026).

Fix #13 — Homepage schema + Berlin focus Done

Structured data (16 Jul 2026)

  • Homepage JSON-LD @graph with linked Organization + WebSite (incl. SearchAction).
  • Includes legal name, logo, address, contact, social sameAs.
  • Validation: Schema.org Validator — 0 errors, 0 warnings on DE / EN / ES homepages.

Berlin-first homepage content (17 Jul 2026)

  • Full-width Berlin hero with localized H1 and short introduction.
  • Primary sections lead with Berlin properties, districts (clean district landing URLs), and Berlin projects only.
  • Dubai / Cyprus kept secondary under “Weitere internationale Immobilienmärkte”, linking to landing pages (not filter search URLs).
  • Compact interactive map under the search bar as visual feedback; defaults to Germany → Berlin.

Fix #14 — Over-optimised district link blocks Done

  • Removed long cross-district link dumps and “Indexierung”-style SEO copy.
  • Replaced with a curated block of ~5 useful links (Berlin hub + district search + nearby districts).
  • Applied to shared district template, dedicated district pages, and the Berlin hub.
  • Validation: Spot-checked Charlottenburg, Moabit, Neukölln, Spandau, and DE Berlin hub (17 Jul 2026).

Fix #15 — Contact form tracking Done

  • GA4 event: contact_form_submit (property G-6PL29347V3).
  • Fires only after a successful server response; never on page load or failed submit.
  • Also fixed a page-view double-count bug in analytics.js (duplicate gtag('config') on load). Reported page views may drop vs prior baselines — that is the double-count removal, not lost traffic.
  • Params: form_type (contact_form, seller_form, property_contact, project_contact, berlin_investor_strategy_form), plus property_id / project_id where relevant.
  • Live validation (20 Jul 2026) on /contact:
    • Page load: contact_form_submit = 0; gtag config = 1
    • After one successful submit: exactly 1 × contact_form_submit with form_type: contact_form
  • Request for Adi: Please mark contact_form_submit as a key event (conversion) in GA4 Admin → Events, and optionally confirm once in Realtime / DebugView.
